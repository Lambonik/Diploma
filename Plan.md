# Перечень используемых инструментов с обоснованием выбора
-Git и GitHub — хранение кода, в том числе автотестов;  
-JUnit5 — наиболее широко используемая среда тестирования для приложений Java;  
-Java 11 — т.к. тестируемый продукт работает на этом языке;  
-Gradle — Gradle более мощный инструмент, чем Maven.

# Перечень и описание возможных рисков при автоматизации
1) Нет требований от заказчика - можно написать много тестов, а важный для заказчика функционал не будет протестирован.
2) Большое время прогонов тестов при усложнении логики тестов.
3) Т.к. у элементов нет test-id, то при изменении интерфейса приложения, тесты могут перестать работать.
4) Ошибки в самих тестах, а будет выглядеть как ошибка ПО.
